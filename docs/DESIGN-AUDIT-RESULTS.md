## Accessibility Audit (Web Design Guidelines)

### Systemic Issues (fix ALL of these)

1. **40+ icon buttons missing aria-label** — every SlidersHorizontal, MoreVertical, X, Star, Trash2, Plus, ChevronLeft/Right button across all 17 files
2. **Zero aria-live regions** — toasts, error messages, status updates invisible to screen readers
3. **Sidebar navigation uses `<button onClick>` + router.push** — breaks Cmd+click, middle-click, right-click-open-new-tab. Must be `<Link>` components
4. **`outline-none` on inputs overrides focus-visible** — users can't see which input is focused via keyboard
5. **No skip-to-content link** — keyboard users must tab through entire sidebar on every page
6. **Touch targets 24-32px** (minimum should be 44px) — icon buttons, checkboxes, sidebar controls
7. **Modals lack role="dialog" + focus trap + Escape key** — time-block picker, delete confirmation, shortcuts overlay
8. **URL state not synced** — tabs/filters/view-modes in useState, not URL params (breaks deep-linking, back button)
9. **No `<meta name="theme-color">`** — browser chrome doesn't match app theme

### Fix Priority

**Batch 1 (30 min): aria-labels** — add to every icon button
**Batch 2 (30 min): Link components** — replace button+router.push with Next.js Link in sidebar + list rows
**Batch 3 (20 min): skip-link + focus fix** — add skip nav, fix outline-none with focus-visible:ring
**Batch 4 (20 min): modals** — add role="dialog", aria-modal, focus trap, Escape key
**Batch 5 (15 min): aria-live + theme-color** — wrap toasts in aria-live, add meta tag

## Visual Design Audit (Frontend Design Skill)

### Current State

LAIF reads as "competent but generic" — the visual language is functional but doesn't create a memorable impression:

- **Inter everywhere** — no typographic personality
- **Flat solid backgrounds** — no atmosphere, no depth
- **Identical page templates** — h1 + banner + list, repeated 16 times
- **No spatial surprises** — rigid grid, no asymmetry, no overlap
- **No hero moments** — no page load that makes someone go "wow"
- **Themes are just color swaps** — same structure, different hex values

### TOP 15 Visual Improvements (ranked by impact)

1. **Typography: Add a display font** — Use Source Serif 4 (already loaded!) for page titles (Inbox, Today, Tasks, Calendar, Focus, Matrix, Habits) at 40px weight 800 with -0.03em tracking. Inter stays for body. This alone transforms the feel.

2. **Login page: hero moment** — Full-bleed gradient background (not flat #0E0E12), floating glass card with subtle backdrop-blur, animated logo on entrance.

3. **Sidebar: depth + polish** — Add a 1px inner-shadow or subtle gradient to distinguish from main pane. Accent-colored dot on active item instead of just a 2px bar.

4. **Focus screen Aurora theme: make it stunning** — The Aurora theme needs actual animated gradients, not just CSS variables. Add a real conic-gradient animation, mesh gradient overlay, and glow halo.

5. **Page transitions** — Cross-fade + subtle slide on route changes. Currently instant/jarring. Use Framer Motion layout animations.

6. **Card depth** — Habit cards, stat cards, list rows currently look flat. Add subtle shadow on hover (not border change) for tactile feel.

7. **Empty states** — Current empty states are plain text. Add subtle illustrations (SVG line art) or at minimum a decorative element.

8. **Calendar blocks: visual polish** — Rounded corners + subtle left-border accent color + shadow on drag. Currently flat rectangles.

9. **Onboarding: dramatic background** — The spec says dusk gradient with moon photo. Currently just a CSS gradient. Add the atmospheric image.

10. **Capacity bar: gradient fill** — Currently a flat color bar. Add a subtle gradient (light-to-dark across the fill) for visual interest.

11. **Checkbox completion: more satisfying** — Add a brief scale overshoot (1.0 → 1.3 → 1.0) with a subtle particle burst, not just fill.

12. **Streak celebrations: actual confetti** — The StreakCelebration component exists but needs real particle physics (not just opacity-fading divs).

13. **Theme preview dots** — Settings theme picker shows flat colored dots. Show mini-previews of actual UI in each theme (tiny sidebar + content + artwork snapshot).

14. **Number animations in stats** — Profile stats show static numbers. Add countUp animation on mount (0 → actual value over 500ms).

15. **Artwork pane: curated, not random** — Currently uses Unsplash URLs that may 404. Bundle 6-8 curated images as local assets. Change daily, match theme mood.
